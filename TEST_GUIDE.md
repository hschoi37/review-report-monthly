# 로컬 테스트 가이드

## 🎯 테스트 현황

✅ **프론트엔드**: 정상 작동 중 (http://localhost:3000)
⚠️ **Python API**: Vercel 배포 환경에서만 작동

## 📊 테스트 옵션

### 옵션 1: Vercel Dev 환경 (추천 ⭐)

Python API를 포함한 전체 기능을 로컬에서 테스트할 수 있습니다.

```bash
# Vercel CLI 설치
npm i -g vercel

# Vercel 개발 서버 실행
vercel dev
```

그 다음 브라우저에서 표시된 URL로 접속 (보통 http://localhost:3000)

**장점:**
- 실제 배포 환경과 동일
- Python API 완전 작동
- 가장 정확한 테스트

### 옵션 2: 빠른 Vercel 배포 테스트

바로 Vercel에 배포해서 테스트합니다.

```bash
# Vercel에 배포 (프리뷰)
vercel

# 결과 URL로 접속해서 테스트
```

**장점:**
- 가장 빠름
- 실제 운영 환경 테스트
- 즉시 공유 가능

### 옵션 3: 프론트엔드만 테스트

현재 실행 중인 개발 서버로 UI/UX만 확인합니다.

**확인 가능한 항목:**
- ✅ 디자인 및 레이아웃
- ✅ 반응형 동작
- ✅ 애니메이션
- ❌ 파일 업로드 (API 필요)
- ❌ AI 분석 (API 필요)

## ⚠️ 주의사항

1. **API 키 필요**: OpenAI API 키가 있어야 AI 분석이 작동합니다
2. **Python 패키지**: `vercel dev` 실행 시 자동으로 설치됩니다
3. **파일 크기**: Vercel 제한 4.5MB (무료 플랜)

## 🐛 문제 해결

### Vercel Dev가 작동하지 않는 경우

```bash
# Python 패키지 수동 설치
cd api
pip install -r requirements.txt
cd ..

# Vercel dev 재시도
vercel dev
```

### API 키 오류

- OpenAI API 키 형식: `sk-...`
- 키가 활성화되어 있는지 확인
- 사용 한도 확인

### 파일 업로드 오류

- 파일 형식: .xlsx 또는 .xls
- 파일 크기: 4.5MB 이하
- 필수 컬럼 포함 확인
